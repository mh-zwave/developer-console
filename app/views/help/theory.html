The Z-Way Server offers a connection between the Z-Wave-wireless-protocol and Users. The Hardware Independent Server-Software is mostly Developed in JavaScript.



The Z-Way Home Automation Software consists out of 3 parts:

  • User-Interface
  • JSON-API (or HTTP-API)
  • JavaScript Server

    • The Z-Way Z-Wave Core is linking the Z-Wave Transceiver and the Z-Wave Device API with the Z-Way JavaScript Core, this is condition a to communicate with Z-Wave Compatible Hardware. Depending on this the Z-Wave Device API delivers a HTTP-Interface, which is used by the Expert UI. The Expert UI is requires to include devices and adjust their Parameters, that means to manipulate the Devices directly. There is also an C-Library with the same functionality.

    But the until now described components not offering any Kind of automation. But there is also an Z-way JavaScript Core who Extends the Z-Way Device API. There is the Place to define Modules to realize Automation. There are some Modules already pre-installed at the System. A Module Consists always out of the same parts, a description of the interface and the JavaScript logic (see. Structure).

    At the Module Continuum Z-way take another step of abstraction for the devices. For example, a multifunctional sensor with 3 functions, is mapped to 3 different virtual devices.

    The virtual devices can only be one of the 4 different Types:


  • Binary Sensor
  • Multilevel Sensor
  • Binary Switch
  • Multilevel Switch

    • The Smart Home UI offers a graphical Interface, who allows to instantiate and configure the modules. Because of the abstraction with JavaScript it is possible, with the accordingly 3rd Party Core, using Other wireless-protocols beside Z-Wave documented at.
