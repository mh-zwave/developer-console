Z-Way supporting to develop own modules in JavaScript, to extend the existing functionality. The Z-Way-server works internal also with JavaScript.

The Z-Way-Server Files can be found in

  • .../automation/classes/

    folder at the Z-Way installation directory and the internal already existing apps can be found at the

  • .../automation/modules/

    folder. Own Modules have to be Placed at

  • .../automation/userModules/.

    • A module, also called app or application in some cases, is a construction plan of a functional part of the Z-Way-Server, who can be instantiated multiple times. Every instance can have his own individual configuration, for example his own virtual Devices or other Parameters picture 1: App Config.


    The result of instantiation (called “Add App” at the User Interface and realized inside the Module with the createInstance({...}) function) ends with on new instance of the Module. A Module has the possibility to create virtual devices during the instantiation process. If this happens there will be shown one Element per instance on the UI picture 2: Instance on UI and it is possible to access this device at the ZAutomation API code 1.


    A module consists out of JavaScript-, JSON- and Text-Files and has to be added to the (Service/automation/userModules/) folder. At this Folder every Module has its own folder and a similar inner structure.

    The folder name sets the label of the modul. The files from the Example Structure have to exist, for the correct loading of the modul at server start. It is necessary to restart the server after every modul change to apply them. In the following a short explaination of the most important modulparts.

  • htdocs/

    folder contains public accessable files for example the Icon, which will be used by the Webinterface, icon sjould be max 128 x 128px and a size of 200kb

  • lang/

    folder contains the individual language files for the user interface in JSON format for example the en.json for the English language and the de.json for the German language

  • index.js

    file contains the prgrammlogic see Operationlogic

  • module.json

    file contains the declaration of the User Interface of the nodul see Interface Logic beside the file- and folder structur there are several rules who have to be correct, for the successful loading of the Modul descripted in the core chapter of operation Logic
